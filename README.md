# 556 Low Code / No Code — Team Onboarding Guide

The team's onboarding guide for new members joining the 556 Low Code / No Code team and starting to use Claude Code.

A single-page, interactive guide that explains:

- What Claude Code is and how we use it
- When to reach for which LLM (build time vs runtime: Claude Code, Anthropic API, Azure OpenAI, Gemini)
- The setup essentials — terminals, install, `--dangerously-skip-permissions`, `claude --continue`, ESC, image paste
- The `CLAUDE.md` file and why it's the most important file in any project
- Skills, our [plugin marketplace](https://github.com/556LowCodeNoCode/Skills), and how to contribute new skills
- GSD (Get Shit Done) — the optional project-management framework
- Three ways to drive a project: plain CC, `/team`, or GSD
- GitHub basics for non-developers (commits, push, PRs, issues — driven through Claude)
- A cheat sheet of common commands

Designed for colleagues with **product / business-analyst backgrounds** — no coding experience required.

## Read it

The hosted version (org members only) is on GitHub Pages — see the repo's About panel for the link, or visit `https://556lowcodenocode.github.io/Onboarding/`.

You can also clone the repo and open `index.html` directly in a browser.

## Edit it

The guide is a single self-contained `index.html` — HTML, CSS, and a tiny bit of JavaScript, no build step. Edit it, commit, push, and GitHub Pages redeploys automatically (~1 minute).

Improvements welcome — open a PR.
